\section{Contexto y Base Te\'orica}
\subsection{Resumen ejecutivo}
Este documento organiza la gesti\'on del proyecto con Scrum y deja una base corta, formal y modular para Overleaf. El foco es el MVP de una plataforma web de mascotas 3D.

\subsection{Visi\'on del producto}
La visi\'on es construir una plataforma donde una persona usuaria pueda registrarse, iniciar sesi\'on, gestionar sus mascotas y visualizar modelos 3D base de animales.

\subsection{Alcance del MVP}
\begin{itemize}[leftmargin=*]
    \item Registro e inicio de sesi\'on.
    \item Alta y consulta de mascotas por usuario.
    \item Visualizaci\'on de modelos 3D base.
    \item Persistencia y trazabilidad funcional.
\end{itemize}

\subsection{Fuera de alcance inmediato}
\begin{itemize}[leftmargin=*]
    \item Personalizaci\'on avanzada de modelos 3D.
    \item Panel administrativo avanzado.
    \item Integraci\'on externa con cat\'alogos de terceros.
\end{itemize}

\subsection{Fundamento Scrum}
Scrum es un marco ligero que ayuda a generar valor mediante soluciones adaptativas para problemas complejos \cite{schwaber2020}. Se apoya en transparencia, inspecci\'on y adaptaci\'on.

\subsection{Valores Scrum}
Los cinco valores son compromiso, foco, apertura, respeto y coraje \cite{schwaber2020}. En este proyecto se usan como gu\'ia para la colaboraci\'on y la entrega incremental.

\subsection{Product Goal}
Entregar una plataforma funcional donde una persona usuaria pueda crear su cuenta, iniciar sesi\'on, registrar mascotas y visualizar modelos 3D base de animales.
