\section{Artefactos y Compromisos}
Scrum define tres artefactos formales y un compromiso por cada uno: Product Backlog con Product Goal, Sprint Backlog con Sprint Goal e Incremento con Definition of Done \cite{schwaber2020}.

\subsection{Product Backlog}
Es la lista ordenada y emergente de todo lo que se necesita para mejorar el producto.

\subsection{Sprint Backlog}
Es el conjunto de PBIs seleccionados para el Sprint y el plan de entrega de los Developers.

\subsection{Incremento}
Es el resultado usable y verificado del Sprint. Si no cumple la Definition of Done, no cuenta como incremento.

\subsection{Compromisos}
\begin{itemize}[leftmargin=*]
    \item Product Backlog $
ightarrow$ Product Goal.
    \item Sprint Backlog $
ightarrow$ Sprint Goal.
    \item Incremento $
ightarrow$ Definition of Done.
\end{itemize}

\subsection{Definition of Done}
\begin{itemize}[leftmargin=*]
    \item C\'odigo integrado y sin errores relevantes.
    \item Pruebas b\'asicas ejecutadas.
    \item Criterios de aceptaci\'on cumplidos.
    \item Sin fallos visibles de compilaci\'on, lint o tipado.
\end{itemize}
